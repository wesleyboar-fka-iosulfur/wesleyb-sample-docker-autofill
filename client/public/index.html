What could we do with the information?

If <datalist> were not used to filter the data (the project requirement to filter seems to be solved, most efficiently, by using this element), then we may make an API call… Here are other solutions that would "use" the data:

  • Use a submit button and API call or silent onChange API call (but, it is more efficient to filter via client-side JavaScript).
  • Have a silent onChange filter perform by client-side JavaScript (but, why do that if the browser—or polyfill—can do the work).
  • Have a form submission (or button "press") trigger a filter perform by client-side JavaScript (still, seems better to rely on a standard HTML element).
  • A benefit of not using the <datalist> element would be more freedom to style the filtered list. But, even then, a standard—and most appropriate—HTML element should be the (hidden) backbone of the functionality.
  • If one builds stylable elements to visually—but not functionally—replace the <datalist>, then all accessibility afforded by the browser would need to be replicated. Is that worth the effort compared to limited styling of widget?
